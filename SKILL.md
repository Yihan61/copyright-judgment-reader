# 民事判决书阅读与结构化提取

## 概述

本技能用于阅读民事判决书 PDF 文件，提取关键信息并以结构化方式呈现。专精于知识产权/版权类案件，同时适用于一般民事判决书。支持一审、二审及再审判决书。

## 触发条件

当用户提供民事判决书 PDF 文件并提出以下需求时触发：
- "帮我读一下这份判决书"
- "提取判决书的关键信息"
- "总结这份判决"
- "分析这份判决"
- 或以任何方式提供判决书 PDF 并要求处理

## 核心原则

### 原则一：完整阅读全文

判决书必须从第一页读取到最后一页，不允许仅读取部分页面后即进行总结或提取。使用 `pdfplumber` 提取全文，确认页数后再进行后续分析。

### 原则二：严格区分审级（最关键）

**这是最容易出错的地方，必须高度警惕。**

对于二审判决书，全文中会出现大量"一审"相关内容，这些是上诉审法院对一审情况的叙述和转述，并非二审法院自身的认定。提取时必须：

1. **明确标注信息来源审级**：每一条提取信息均须标注属于"一审认定"还是"二审认定"
2. **区分"本院认为"的归属**：二审判决书中，"一审法院认为"或"一审法院认定"后跟的论述是一审的观点；"本院认为"后跟的论述是二审的观点
3. **区分"本院查明"的归属**：二审判决书中，"一审法院查明事实如下"后跟的是一审认定的案件事实；"二审查明如下事实"或"本院查明"后跟的是二审补充查明的事实
4. **区分判决结果**：一审判决结果和二审判决结果是不同的，不可混淆
5. **上诉请求和答辩意见属于二审程序**：上诉人的上诉请求和被上诉人的答辩意见，是二审阶段的内容

### 原则三：结论先行，细节在后

输出信息时，先给出结构化摘要（表格形式），再展开各部分的详细内容。用户可通过进一步指令获取特定部分的深度分析。

## 判决书类型识别

首先通过案号判断审级：
- 案号含"初"字（如"渝01民初54号"）：一审判决书
- 案号含"终"字（如"渝民终35号"）：二审判决书
- 案号含"再"字（如"民再XX号"）：再审判决书
- 文首标注"XX人民法院民事判决书"：一审判决书
- 文首标注"XX高级人民法院民事判决书"且案号含"终"：二审判决书

同时关注判决书结尾：
- "本判决为终审判决"：二审判决
- "如不服本判决，可在XX日内上诉"：一审判决

## 判决书标准结构

### 一审判决书结构

```
一、当事人信息（原告、被告、第三人）
二、原告诉讼请求及事实与理由
三、被告答辩意见
四、法院查明事实
五、法院认为（争议焦点分析）
六、判决结果
七、诉讼费用负担
八、审判人员、日期
```

### 二审判决书结构

```
一、当事人信息（上诉人、被上诉人）
二、一审基本案情概述（简述一审审理情况）
三、上诉人上诉请求及事实与理由
四、被上诉人答辩意见
五、一审法院查明事实（完整转述）
六、一审法院认为（争议焦点分析及一审裁判理由）
七、一审判决结果
八、二审证据（新证据及质证）
九、二审查明事实（补充查明）
十、二审法院认为（逐条分析争议焦点）
十一、二审判决结果（维持/改判/撤销）
十二、诉讼费用负担
十三、审判人员、日期
```

### 第三部分：侵权行为认定（核心部分）

此部分为重点，按审级分别提取，每个审级下按以下子结构组织：

#### 一审法院认定

**1. 权属认定**
- 权利人身份、授权链条

**2. 侵权行为定性**
- 是否构成直接侵权：结论 + 理由
- 是否构成帮助侵权：结论 + 理由
  - 明知/应知认定（详细论述）
  - 必要措施认定（详细论述）

**3. 不正当竞争认定（如涉及）**
- 结论 + 理由

#### 二审法院认定（仅二审判决书）

**1. 权属认定**
- 是否维持/变更一审认定 + 理由

**2. 侵权行为定性**
- 直接侵权：是否维持/变更一审认定 + 理由
- 帮助侵权：是否维持/变更一审认定 + 理由
  - 明知/应知认定（二审观点，详细论述）
  - 必要措施认定（二审观点，详细论述）
  - 注：二审是否有新的认定或对一审的纠正

**3. 不正当竞争认定**
- 是否维持/变更一审认定 + 理由

**4. 惩罚性赔偿认定（如涉及）**

## 常见判断规则与注意事项

### 识别审级边界的关键词

**一审内容边界关键词（在二审判决书中）：**
- "一审法院查明事实如下" → 后面是一审认定的事实
- "一审法院认为" / "一审法院认定" → 后面是一审的裁判理由
- "一审法院依照……判决如下" → 后面是一审判决结果
- "一审案件受理费" → 一审诉讼费用

**二审内容边界关键词：**
- "二审期间，XX公司提交以下证据" → 二审新证据
- "二审查明如下事实" / "本院另查明" → 二审补充查明的事实
- "本院认为" → 二审法院的裁判观点（注意排除"一审法院认为"的干扰）
- "本院依照……判决如下" → 二审判决结果
- "本判决为终审判决" → 确认为二审判决

### 知识产权类案件特别关注点

1. **权属链条**：作品署名 → 著作权人 → 授权链条 → 原告权利来源，须完整梳理
2. **侵权行为分类**：
   - 直接提供作品（服务器存储、直接传播）
   - 帮助侵权（链接、搜索、技术支持）
   - 教唆侵权（诱导、鼓励）
3. **平台责任认定要素**：
   - "明知或应知"的具体认定标准和事实依据
   - "必要措施"的具体内容和法院的评价标准
   - "通知-删除"规则的适用
   - 过错认定中是否考虑了平台的技术能力、商业模式
4. **损害赔偿计算**：
   - 原告的计算方式和依据
   - 法院采用的计算方式
   - 惩罚性赔偿的适用条件是否满足
   - 合理开支的认定

### 输出规范

1. 对话输出时使用 Markdown 格式，表格呈现对比信息
2. 使用引用块（`>`）标注法院原文中的关键论述
3. 对于争议焦点的分析，先用一句话概括结论，再展开论述逻辑
4. 保持客观，忠实于判决书原文，不添加主观评价
5. 如需生成独立文件（Markdown/Word），根据用户指令使用 docx skill 或直接写入文件

## 执行流程

1. **接收判决书**：用户提供 PDF 文件路径
2. **全文提取**：使用 `pdfplumber` 读取全文，确认总页数
3. **类型识别**：通过案号和判决书格式判断审级
4. **全文分析**：逐段阅读全文，按上述模板提取信息
5. **结构化输出**：按模板格式呈现提取结果
6. **等待指令**：根据用户进一步指令，可深入分析特定部分或生成独立文件

## 错误防范

- **不要将一审法院的认定当作二审法院的认定**：仔细辨别"一审法院认为"和"本院认为"
- **不要将当事人的主张当作法院的认定**：当事人主张的事实和法院查明的事实可能不同
- **不要将上诉请求当作一审判决结果**：上诉请求是对一审判决的不服，不是一审判决本身
- **不要遗漏二审判决对一审判决的变更**：二审判决可能撤销、变更一审判决的部分或全部内容
