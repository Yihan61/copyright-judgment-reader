# judgment-reader

面向民事判决书的结构化阅读技能，重点适用于知识产权、著作权及平台责任相关案件。

该技能用于在 Claude Code 中读取判决书 PDF，全书分析后输出结构化摘要，尤其强调严格区分一审、二审、再审各自的事实认定、裁判理由与判决结果，避免将不同审级内容混淆。

## 功能特点

- 支持一审、二审、再审民事判决书阅读
- 要求全文读取，不允许只读局部页面后直接总结
- 自动提取基础信息、诉讼请求、答辩要点、侵权认定、赔偿金额、法律依据、判决结果
- 对二审案件重点区分“一审法院认为”与“本院认为”
- 适合著作权、信息网络传播权、平台帮助侵权、不正当竞争等案件分析

## 适用场景

当用户提供民事判决书 PDF 并提出以下需求时适用：

- 帮我读一下这份判决书
- 提取判决书关键信息
- 总结这份判决
- 分析判决中的侵权认定逻辑
- 对比一审和二审裁判理由

## 核心方法

### 1. 全文读取

判决书必须从第一页读到最后一页，再进行信息提取与归纳。

### 2. 严格区分审级

这是该技能最重要的约束，尤其适用于二审判决书：

- “一审法院认为”属于一审裁判理由
- “本院认为”通常属于当前审级法院观点
- “一审法院查明事实如下”属于一审事实
- “本院查明”或“二审查明如下事实”属于二审补充事实
- 一审判决结果与二审判决结果必须分别提取

### 3. 结构化输出

默认先输出摘要表格，再展开诉讼请求、侵权认定、赔偿对比、判决结果和法律依据。

## 输出内容

技能通常会整理以下内容：

- 案号、法院、审级、案由
- 当事人信息
- 原告或上诉人诉讼请求
- 被告或被上诉人答辩要点
- 一审认定与二审认定
- 直接侵权、帮助侵权、不正当竞争认定
- 明知或应知、必要措施、通知删除等论证逻辑
- 赔偿金额与诉讼费用
- 判决主文与法律依据

## 使用方式

在 Claude Code 中调用：

```text
/judgment-reader
```

然后提供判决书 PDF 文件路径，或直接上传 PDF，并提出你的分析需求。

## 示例请求

```text
请帮我提取这份判决书的结构化信息，并重点分析二审对帮助侵权的认定。
```

```text
请总结这份著作权案件判决，并区分一审和二审的赔偿理由。
```
